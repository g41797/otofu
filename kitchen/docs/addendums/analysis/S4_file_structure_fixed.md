## Package: `types`

Shared definitions. No layer. No logic. No procedures.
All types in this package are value types or enums. No heap allocation, no imports.

---

### `types/opcodes.odin`

**Purpose:** Wire protocol opcode enum and associated packed flag types. These exactly match the tofu wire protocol (RULE-3). No otofu-internal "command" opcodes appear here — commands are carried as wire messages with appropriate opcodes and TextHeaders.

**Types:**

```odin
// Wire protocol opcode. 10 values matching tofu exactly.
// Fits in 4 bits (u4) inside ProtoFields.
OpCode :: enum(u4) {
    Request       = 0,   // Application data — bidirectional
    Response      = 1,   // Application data — bidirectional
    Signal        = 2,   // Application data — bidirectional (one-way, no response expected)
    HelloRequest  = 3,   // Client → Server: initiate connection handshake
    HelloResponse = 4,   // Server → Client: handshake accepted
    ByeRequest    = 5,   // Either side: begin graceful close
    ByeResponse   = 6,   // Responding side: close acknowledged
    ByeSignal     = 7,   // Engine → peer: force close signal
    WelcomeRequest  = 8, // Client → Server: post-hello completion
    WelcomeResponse = 9, // Server → Client: channel open confirmation
}

// 1-bit origin flag embedded in ProtoFields.
// .application: message from application layer
// .engine:       message generated by otofu engine internals
OriginFlag :: enum(u1) {
    application = 0,
    engine      = 1,
}

// 1-bit more-messages flag embedded in ProtoFields.
MoreMessagesFlag :: enum(u1) {
    last = 0,
    more = 1,
}

// Packed 1-byte protocol fields embedded in BinaryHeader at wire level.
// Total 8 bits: opCode(4) + origin(1) + more(1) + _internalA(1) + _internalB(1)
ProtoFields :: bit_field u8 {
    opCode:    OpCode          | 4,
    origin:    OriginFlag      | 1,
    more:      MoreMessagesFlag | 1,
    _internalA: u1             | 1,
    _internalB: u1             | 1,
}
```

**Note on addresses:** Connect and listen addresses are NOT opcodes. They are encoded as TextHeaders (`~connect_to`, `~listen_on`) in HelloRequest/WelcomeRequest wire messages. No OpCode named Connect, Listen, Close, Drain, or Channel_Closed exists.

**Relationships:** Referenced by `message/message.odin` (BinaryHeader), `protocol/protocol.odin` (dispatch switch). No intra-package dependency.

### `types/status.odin`

**Purpose:** Wire-level status byte type (RULE-1). `Ampe_Status` is an `enum u8` — its numeric values travel in `BinaryHeader.status` between processes and machines and must match tofu exactly. In Odin, `enum u8` serves as both the wire byte and the procedure return value directly; no separate error type is needed (unlike Zig's `AmpeError` which existed only because Zig error sets have no guaranteed numeric values).

```odin
Ampe_Status :: enum u8 {
    success = 0,
    not_implemented_yet,
    wrong_address,
    not_allowed,
    null_message,
    invalid_message,
    invalid_op_code,
    invalid_headers_len,
    invalid_body_len,
    invalid_more_usage,
    invalid_channel_number,
    invalid_mchn_group,
    invalid_message_id,
    invalid_address,
    uds_path_not_found,
    notification_disabled,
    notification_failed,
    peer_disconnected,
    communication_failed,
    pool_empty,
    allocation_failed,
    receiver_update,
    channel_closed,
    shutdown_started,
    connect_failed,
    listen_failed,
    accept_failed,
    send_failed,
    recv_failed,
    setsockopt_failed,
    processing_failed,
    unknown_error,
}

// Wire conversion (u8 ↔ Ampe_Status).
// Used when reading BinaryHeader.status from a received frame.
Status_From_Raw :: proc(rs: u8) -> Ampe_Status
Status_To_Raw   :: proc(st: Ampe_Status) -> u8
```

**Not ported from tofu:** `AmpeError` (Zig error set), `raw_to_error`, `status_to_error`, `errorToStatus` — these exist only because Zig errors lack guaranteed numeric values. In Odin, `Ampe_Status` is used directly in return types everywhere.

**Replaces:** `Engine_Error` (13-value enum previously in `engine.odin`) — removed; all API procedures now return `Ampe_Status`.

**Relationships:** Used by every package that returns a status to the caller. `BinaryHeader.status` field type is `Ampe_Status`. `engine.odin` procedure signatures return `Ampe_Status`.

## Package: `message`

Top-level public package. Contains the `Message` type, wire types, TextHeaders, intrusive queue, `Address` type and behavior, and helpers. Application code imports this package directly. All otofu internal packages also import it.

`message.zig` (837 lines) + `address.zig` map to **six files** within the `message/` package. All six files share the same `package message` declaration. Client code imports the package once and accesses all exported identifiers regardless of which file they live in.

---

### `message/binary_header.odin`

**Purpose:** Wire-format types: `ChannelNumber`, `MessageID`, special channel number constants, `BinaryHeader` packed struct and its encode/decode procedures.

**Types (RULE-1 — wire format preserved exactly from tofu):**

```odin
// Imports: types (for OpCode, ProtoFields, OriginFlag, MoreMessagesFlag)

ChannelNumber :: u16
MessageID     :: u64

SPECIAL_MIN_CHANNEL :: min(ChannelNumber)
SPECIAL_MAX_CHANNEL :: max(ChannelNumber)
FRAME_HEADER_SIZE   :: 16

// 16-byte packed big-endian binary header.
// This is the EXACT wire format transmitted over TCP/UDS.
// Field order and sizes must not change — protocol compatibility depends on it.
BinaryHeader :: struct #packed {
    channel_number: u16be,       // channel correlation token
    proto:          types.ProtoFields, // packed u8: opCode(4), origin(1), more(1), _A(1), _B(1)
    status:         u8,          // operation status code
    message_id:     u64be,       // request-response correlation
    thl:            u16be,       // text headers length in bytes (engine-internal)
    bl:             u16be,       // body length in bytes (engine-internal)
}
#assert(size_of(BinaryHeader) == FRAME_HEADER_SIZE)
```

**Procedures:**

```odin
// Serialize BinaryHeader to 16-byte big-endian wire buffer (handles LE platforms).
BH_To_Wire   :: proc(bh: ^BinaryHeader, buf: *[FRAME_HEADER_SIZE]u8)

// Deserialize 16-byte big-endian wire buffer into BinaryHeader (handles LE platforms).
BH_From_Wire :: proc(bh: ^BinaryHeader, buf: *[FRAME_HEADER_SIZE]u8)

BH_Init  :: proc(op: types.OpCode) -> BinaryHeader
BH_Clean :: proc(bh: ^BinaryHeader)
BH_Dump  :: proc(bh: ^BinaryHeader, label: string)   // debug only
```

### `message/text_headers.odin`

**Purpose:** TextHeaders encoding/decoding in HTTP-style "name: value\r\n" format (RULE-1).

**Types:**

Appendable :: struct {

Text_Header :: struct {

### `message/message.odin`

**Purpose:** The `Message` type — primary unit of data exchange. Embeds PolyNode at offset 0 (RULE-5). Provides lifecycle procedures and application-facing field accessors. Also defines `MessageType`, `MessageRole`, `Trigger` (used by `check_and_prepare`).

**Types:**

```odin
// Imports: types, matryoshka

MESSAGE_ID :: 1   // PolyNode discriminator constant for Message

MessageType :: enum(u8) { regular = 0, welcome = 1, hello = 2, bye = 3 }
MessageRole :: enum(u8) { request = 0, response = 1, signal = 2 }
Trigger     :: enum(u8) { off = 0, on = 1 }

// Primary unit of message ownership in otofu.
// PolyNode at offset 0 — mandatory (P4 C1).
// RULE-5: PolyNode is internal; application accesses only bh, thdrs, body.
Message :: struct {
    using _node: matryoshka.PolyNode,  // offset 0; id = MESSAGE_ID
    bh:           BinaryHeader,         // 16-byte wire header (embedded; no heap)

```

### `message/queue.odin`

**Purpose:** Single-threaded intrusive FIFO for `*Message`. Odin port of `tofu/src/ampe/IntrusiveQueue.zig` (RULE-8). Matryoshka provides only concurrent Mailbox (with mutex); it has no single-threaded intrusive queue equivalent. Used by `chanmgr/outbound.odin` (per-channel send queue) and reactor scratch collections.

**Design:** `Message` has `PolyNode` at offset 0. `PolyNode` embeds `list.Node` (from `core:container/intrusive/list`) with `prev`/`next` pointers. `Message_Queue` uses these fields directly — zero additional allocation.

**NOT thread-safe.** For concurrent queuing use Matryoshka `Mailbox`.

**Types:**

```odin
Message_Queue :: struct {
    first: ^Message,
    last:  ^Message,
}
```

**Procedures:**

```odin
// Add msg to back of queue (FIFO order).
MQ_Enqueue    :: proc(q: ^Message_Queue, msg: ^Message)

// Add msg to front of queue (priority insertion — used for protocol control messages).
// Mirrors tofu IntrusiveQueue.pushFront.
MQ_Push_Front :: proc(q: ^Message_Queue, msg: ^Message)

// Remove and return front message; nil if empty.
MQ_Dequeue    :: proc(q: ^Message_Queue) -> ^Message

MQ_Empty      :: proc(q: ^Message_Queue) -> bool
MQ_Count      :: proc(q: ^Message_Queue) -> int

// Transfer all items from src into dst (src becomes empty after call).
// Mirrors tofu IntrusiveQueue.move.
MQ_Move       :: proc(src: ^Message_Queue, dst: ^Message_Queue)

// Dequeue and destroy all messages (calls Msg_Destroy on each).
MQ_Clear      :: proc(q: ^Message_Queue)
```

### `message/address.odin`

**Purpose:** Network address type and all address behavior. Full consolidation of tofu's `address.zig`. Lives in `message/` because `Address` is active — its format/parse procedures work with `TextHeaders`, and no package needs `Address` without also needing `Message`. Moving it here eliminates the former split between `types/address.odin` (type only) and a separate bridge file (behavior only).

**Types (RULE-7 — all four variants preserved):**

```odin
IP_Version :: enum { V4, V6 }

// TCP connect address (client side — reactor calls connect())
TCP_Client_Address :: struct {
    host:    string,
    port:    u16,
    version: IP_Version,
}

// TCP listen address (server side — reactor calls listen()/accept())
TCP_Server_Address :: struct {
    host:    string,   // "" = all interfaces / INADDR_ANY
    port:    u16,
    version: IP_Version,
}

// UDS connect address (client side — reactor calls connect())
UDS_Client_Address :: struct {
    path: string,   // e.g. "/tmp/7099.port"
}

// UDS listen address (server side — reactor calls listen()/accept())
UDS_Server_Address :: struct {
    path: string,
}

// Tagged union — variant determines reactor dispatch (connect vs listen/accept).
Address :: union {
    TCP_Client_Address,
    TCP_Server_Address,
    UDS_Client_Address,
    UDS_Server_Address,
}

// TextHeader names that carry the address on the wire (RULE-1).
ConnectToHeader :: "~connect_to"   // HelloRequest TextHeaders
ListenOnHeader  :: "~listen_on"    // WelcomeRequest TextHeaders
```

**Active behavior (RULE-6 — format/parse; header integration):**

```odin
// Serialize Address to wire string for a TextHeader value.
// Format matches tofu exactly:
//   TCP client/server: "tcp|127.0.0.1|7099"
//   UDS client/server: "uds|/tmp/7099.port"
// Returns false if addr is nil variant.
Addr_Format :: proc(addr: Address, buf: []u8) -> (written: string, ok: bool)

// Reconstruct Address from a TextHeader value string.
// Prefix determines variant: "tcp|" → TCP_Client_Address, "uds|" → UDS_Client_Address, etc.
Addr_Parse :: proc(s: string) -> (addr: Address, ok: bool)

// Dispatch helpers — used by reactor/io_dispatch.odin.
Addr_Is_Client :: proc(addr: Address) -> bool   // TCP_Client or UDS_Client → connect()
Addr_Is_Server :: proc(addr: Address) -> bool   // TCP_Server or UDS_Server → listen()/accept()

// Write address into message TextHeaders.
Addr_To_Connect_Header :: proc(addr: Address, msg: ^Message) -> bool
Addr_To_Listen_Header  :: proc(addr: Address, msg: ^Message) -> bool

// Read address from message TextHeaders.
Addr_From_Connect_Header :: proc(msg: ^Message) -> (Address, bool)
Addr_From_Listen_Header  :: proc(msg: ^Message) -> (Address, bool)
```

**No import cycle:** `message/address.odin` is in the `message` package, which already imports `types` (for OpCode etc.). Address types are local to the package — no back-reference to `types` for Address.

**Relationships:** `Address` used by `otofu/engine.odin` (CG_Connect/CG_Listen parameters), `reactor/io_dispatch.odin` (variant dispatch), `protocol/handshake.odin` (header reading).

### `runtime/framer.odin`

**Purpose:** Wire encoding and decoding. Converts a `message.Message` to a byte stream for socket send, and decodes an inbound byte buffer into a `message.Message`. Uses the exact BinaryHeader wire format (big-endian, 16 bytes) followed by TextHeaders bytes and body bytes (RULE-1).

### `otofu/channel_group.odin`

**Purpose:** ChannelGroup handle API. Full tofu ChannelGroup API equivalent (RULE-2), including `CG_Update_Receiver`.

**Types:**

```odin

    id:     types.ChannelGroupId,
    router: ^runtime.Router,   // non-owning reference to engine's Router
}
```

**Public procedures:**
```odin
CG_Create          :: proc(engine: Engine) -> (CG, types.Engine_Error)
CG_Destroy         :: proc(engine: Engine, cg: CG)
CG_Post            :: proc(cg: CG, m: ^matryoshka.MayItem) -> types.Engine_Error
CG_Wait_Receive    :: proc(cg: CG, m: ^matryoshka.MayItem, timeout_ns: u64) -> types.Engine_Error
CG_Update_Receiver :: proc(cg: CG, m: ^matryoshka.MayItem) -> types.Engine_Error
CG_Connect         :: proc(cg: CG, addr: types.Address, m: ^matryoshka.MayItem) -> types.Engine_Error
CG_Listen          :: proc(cg: CG, addr: types.Address, m: ^matryoshka.MayItem) -> types.Engine_Error
```

**`CG_Update_Receiver` semantics:**
- If `m^` is not nil: engine sets `bh.proto.opCode = .Signal`, `bh.proto.origin = .engine`; enqueues to CG outbox; sets `m^ = nil`.
- If `m^` is nil: engine creates a synthetic Signal from `reserved_pool` and enqueues to CG outbox; no Message consumed from caller.
- Returns error if Engine is draining or destroyed.
- Thread-safe: may be called from any thread (not only the receiver thread).

**`CG_Connect` / `CG_Listen` semantics:**
- Encodes the `addr` as a TextHeader (`~connect_to` or `~listen_on`) in the Message.
- Sets `bh.proto.opCode = .HelloRequest` (for connect) or `.WelcomeRequest` (for listen).
- Calls `CG_Post` to deliver to Reactor inbox.

**Relationships:**
- `CG_Post`: calls `runtime.Router_Send_Reactor` then `notifier_notify` (ordering: AI-7).
- `CG_Wait_Receive`: calls `runtime.Router_Wait_App`.
- `CG_Update_Receiver`: calls `runtime.Router_Send_App` directly (bypasses reactor inbox — delivers to CG outbox directly).
- Depends on `engine.odin` for the `Engine` type.

## Constraints Carried Forward

| Constraint | Structural consequence |
|------------|----------------------|
| P4 C1 — PolyNode at offset 0 | `message.Message.poly`, `TC.poly`, `Channel.poly` are the first field via `using` |
| P4 C7 — Reactor must not call `mbox_wait_receive` | `router.odin` exposes `Router_Drain_Inbox` (non-blocking); `Router_Wait_App` is for application threads only |
| P4 V-M3 — Reserved pool is separate | `reserved_pool.odin` is a distinct file and type from `message_pool.odin`; Reserved_Pool is in reactor_state NOT engine_internal |
| P4 V-TC1 — TC not returned to pool while registered | `tc_pool.odin` defines `TC_State`; `reactor.odin` enforces: pool_put only in Phase 8, after deregister |
| P4 V-MAR1 — `mbox_interrupt` not used for wake | `router.odin` uses `Wake_Fn` (socket-pair notify); no `mbox_interrupt` in any file |
| P5 P-01 — No circular import reactor↔otofu | `reactor_state` holds `Engine_Options` by value (copied at startup) and `^types.Engine_State` (pointer to types-level field); no import of `otofu` package |
| P5 P-02 — No circular import reactor↔protocol | `dispatch_context` is lowercase and package-private; protocol receives `^Protocol_Context` only |
| P5 P-03 — Reserved_Pool ownership | `Reserved_Pool` lives in `reactor_state` only; NOT in `engine_internal` |
| P5 P-04 — Channel collection | `reactor_state.channels` is the primary owned collection of all active Channels |
| P6 LI-2 / AI-2 — All dual_map inserts in Phase 5, removes in Phase 8 | `dual_map.odin` is write-only in phases 5/8; `io_dispatch.odin` does not mutate `Dual_Map` |
| P6 LI-8 — inbox send before notify | `router.odin` `Router_Send_Reactor`: `mbox_send` call appears before `wake_fn` call |
| S2 — L4 does not call `timeout_manager` | `protocol/protocol.odin` and `protocol/handshake.odin` have no `Timeout_Manager` field |
| S3 — `chanmgr` is separate from `reactor` | `protocol/protocol.odin` imports `chanmgr` but not `reactor` |
| RULE-1 — Wire format immutable | `message.BinaryHeader` = exact 16-byte tofu wire layout; `message.TextHeaders` = "name: value\r\n" format |
| RULE-4 — UDS support | `platform/socket.odin` handles both TCP and UDS; `types.Address` is a union of TCP_Address and UDS_Address |
